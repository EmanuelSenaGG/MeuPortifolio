Sistema de Mensageria Assíncrona

Solução de comunicação assíncrona entre microsserviços para garantir a entrega de notificações com alta resiliência e escalabilidade.

Sobre o Projeto

Para resolver falhas críticas e garantir a entrega de e-mails em um sistema distribuído, arquitetei e implementei uma solução de mensageria assíncrona. Utilizando RabbitMQ como message broker e .NET Worker Services para o processamento em background, desacoplei a comunicação entre 4 serviços distintos. A nova arquitetura garantiu a entrega de mensagens e introduziu mecanismos robustos de resiliência, como políticas de retry para falhas temporárias e uma dead-letter queue (lixeira) para análise de erros persistentes, eliminando a perda de notificações.

Principais Tecnologias

  • RabbitMQ
  • Worker Services
  • ASPNET Web APIs

Arquitetura e Aprendizados

Evoluí a arquitetura da API de Notificação para resolver gargalos de concorrência e balanceamento de carga. O serviço, que antes enviava e-mails de forma síncrona, foi transformado em um Publisher que direciona as mensagens para filas individuais em um message broker. Essa abordagem permitiu o processamento assíncrono e isolado das notificações, onde cada sistema possui seu próprio Consumer com lógicas de tratamento específicas, garantindo maior escalabilidade e resiliência.
